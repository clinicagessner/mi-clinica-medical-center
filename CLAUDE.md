## Development Rules

### ALWAYS

1. **Server Components by default** - Only `'use client'` when useState/useEffect/event handlers/browser APIs are needed
2. **Direct imports** from shadcn/ui (never barrel imports)
3. **Validate with Zod** before sending data (client + server)
4. **next/image** for all images, with descriptive alt text
5. **Promise.all** for parallel data fetching
6. **Mobile-first** in Tailwind (`w-full md:w-[800px]`)
7. **Loading states** on buttons and forms
8. **CSS variables** for theme colors
9. **next-intl** for all user-facing text (use translation keys)
10. **Reduced motion** support in Framer Motion animations

### NEVER

1. `'use client'` without real need
2. Data waterfalls (sequential fetch when parallel is possible)
3. Barrel imports (`import { Button, Card } from '@/components/ui'`)
4. Hardcode hex colors in classes
5. Use native `<img>` (use next/image)
6. Omit alt text on images
7. Hardcoded text (use next-intl translation keys)
8. `forwardRef` in React 19 (use ref directly as prop)
9. `tailwind.config.ts` in Tailwind v4 (use `@theme inline` in globals.css)
10. Add AI `Co-Authored-By` in commits

## Accessibility (WCAG 2.1 AA)

### Main Rules

- All icon-only buttons need `aria-label`
- Decorative icons must have `aria-hidden="true"`
- Forms with `aria-invalid` and `aria-describedby` for errors
- Visible focus with `outline: 2px solid var(--primary)`
- Minimum contrast text: 4.5:1, icons/UI: 3:1
- Respect `prefers-reduced-motion`
- Modals: focus trap, Escape to close, restore focus on close
